# Git Push Branch

Git の変更を安全に commit / push するためのスキル。

## 使う場面

- 「変更をコミットして push して」
- 「今のブランチを GitHub に上げて」
- 「コミットを作ってリモートに送って」

## 手順

1. `git status` と差分を確認し、何をコミットするか把握する。
2. 機密情報、認証情報、`.env` などの秘匿ファイルが含まれる場合は止めてユーザー確認を取る。
3. 現在のブランチ名を確認する。
4. ブランチが `main` / `staging` / `production` の場合は、commit や push の前にユーザー確認を取る。
5. 変更をコミットする。コミットメッセージは差分から具体的に作る。
6. 現在のブランチを対応するリモートブランチへ push する。
7. push 後、現在のブランチが `main` ではない場合は、GitHub の PR 作成ページへのリンクを表示する。

## 実行ルール

- 既存の未コミット変更は勝手に破棄しない。
- コミット対象が曖昧な場合は、まとめてコミットせずユーザーに確認する。
- push 先は通常 `origin/<current-branch>` を使う。上流ブランチが未設定なら適切に設定して push する。
- PR は作成しない。表示するのは PR 作成ページのリンクだけにする。
- リンクは compare URL など、ユーザーがそのままブラウザで開ける形式で示す。

## 出力

- 作成したコミットメッセージ
- push したブランチ名とリモート名
- `main` 以外なら PR 作成リンク
